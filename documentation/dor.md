# DoR — Definition of Ready

Uma User Story ou Task só pode entrar em sprint quando **todos** os critérios abaixo estiverem atendidos.

---

## User Stories

- [ ] Escrita no formato `Como <usuário>, quero <funcionalidade> para <motivação>` com terminologia do domínio jurídico
- [ ] Campo **Requisito** preenchido com URL do RF correspondente no Confluence
- [ ] Vinculada ao épico correto no Jira
- [ ] Regras de negócio descritas (incluindo comportamentos da IA quando aplicável)
- [ ] Dados envolvidos definidos (campos, tipos, origens, validações)
- [ ] Critérios de aceitação listados no formato `Dado / Quando / Então`
- [ ] Protótipo ou wireframe aprovado pelo PO (ou referenciado no campo Requisito)
- [ ] Estimativa de esforço atribuída via Planning Poker
- [ ] Tasks técnicas criadas, vinculadas e estimadas individualmente
- [ ] Sem dependências externas não resolvidas

## Tasks

- [ ] Título claro e objetivo descrevendo o que será implementado
- [ ] Vinculada à User Story correspondente
- [ ] Camada da aplicação identificada (`server`, `mobile`, `iac`)
- [ ] Estimativa em story points atribuída
- [ ] Sem bloqueadores ativos no momento da entrada em sprint
